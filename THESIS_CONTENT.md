RESEARCH BACKGROUND AND SIGNIFICANCE:

1. Research Objectives:
The main goal of this research is to develop a smart, web-based dashboard for Kubernetes clusters. This system is designed to turn complex data into clear, easy-to-understand information. The project focuses on creating a single platform that combines metrics, logs, and events in one place. Additionally, it aims to use the "Isolation Forest" algorithm to find system problems automatically and create a simple visual layout that helps engineers manage clusters without feeling overwhelmed.

2. Social, Technical, and Academic Background:
From a social perspective, our modern world relies on cloud services for everything from banking to healthcare. When these systems fail, it affects many people's daily lives. Technically, as more companies move to "microservices," the number of containers has exploded. A 2024 survey shows that 96% of organizations now use Kubernetes, making it the industry standard. Academically, current research in "AIOps" (Artificial Intelligence for IT Operations) shows that while we have plenty of data, we lack tools that can analyze it all at once to find the real cause of a problem quickly.

3. Motivation and Problem Solving:
The motivation for this project comes from the fact that current tools are often scattered and hard to use. Engineers often have to jump between different programs to see what is wrong, which wastes time during a crisis. This project solves these problems by: Unifying Data: Bringing everything (logs and metrics) into one screen
