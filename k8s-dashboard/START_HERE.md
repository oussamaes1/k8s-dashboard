## The One Thing You MUST Memorize

### Your Opening Statement (60-90 seconds)

```
"This project presents a monitoring prototype for Kubernetes clusters, 
designed to demonstrate cluster observability techniques through data 
visualization and unsupervised anomaly detection.

The system serves as a DECISION-SUPPORT TOOL that helps human operators 
observe cluster behavior and investigate unusual patterns. It does NOT 
make autonomous decisions or modify the cluster—rather, it aggregates 
information from the Kubernetes API and highlights interesting 
observations for operator review.

The key contributions are:
1. Integration of Kubernetes Python Client with modern web frameworks
2. Demonstration of unsupervised ML application to infrastructure data
3. Exploration of correlation-based analysis for troubleshooting support

The prototype achieves its educational objectives within appropriate scope, 
while honestly acknowledging limitations."
```

**Practice until you can say this naturally, not robotic.**

---

## The Three Phrases You MUST NOT SAY

❌ **Never say:** "AI-powered intelligent system"  
✅ **Say instead:** "Unsupervised anomaly scoring prototype"

❌ **Never say:** "Automatically detects and fixes problems"  
✅ **Say instead:** "Highlights unusual patterns for operator review"

❌ **Never say:** "Production-ready monitoring solution"  
✅ **Say instead:** "Educational proof-of-concept"

## Emergency Backup Plans

### If demo fails:
→ Have offline demo screenshots ready  
→ Show code on screen  
→ Explain what would show if cluster available  

### If you forget an answer:
→ Say "That's a great question. Let me think..."  
→ Take 5-10 seconds  
→ Give honest answer (or "I don't know but would investigate X")  

### If asked something unexpected:
→ Link back to your opening statement  
→ Reference limitations  
→ Admit if outside your scope  

### If nervous:
→ Take deep breath  
→ Remember committee WANTS you to pass  
→ Focus on demonstrating understanding, not perfection

## During Defense: Do's and Don'ts

**DO:**
- ✓ Speak clearly and at normal pace
- ✓ Make eye contact
- ✓ Pause after questions
- ✓ Admit what you don't know
- ✓ Be honest about limitations
- ✓ Show confidence in what you do know
- ✓ Engage with the committee

**DON'T:**
- ✗ Rush to answer
- ✗ Oversell capabilities
- ✗ Be defensive about criticism
- ✗ Make up answers
- ✗ Speak in marketing language
- ✗ Ignore committee comments
- ✗ Act nervous or uncertain about things you know
